# Changelog

All notable changes to Sloth-Playbook-Eido will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/), and this project adheres to [Semantic Versioning](https://semver.org/).

## [1.0.0] - 2026-04-21

### Added
- Release checklist module with 4-level verification (必需/推荐 tags)
- Test report template with 6-chapter framework
- CHANGELOG specification based on Keep a Changelog format
- SemVer version management with upgrade decision tree
- Standard skill repo directory structure specification
- Regression test suite standard with TC-{module}-{seq} numbering
- Git commit conventions (type(scope): subject format)
- Cross-skill integration interface specification for data exchange
- Skill health check module with version status tracking
- Quick reference: 5-minute pre-release checklist
- Three output density modes: Lightning / Standard / Deep Dive
- Intent routing with module-specific trigger words
- Cross-module context passing (e.g. release check auto-carries skill name to test report)
- Reference documentation: `references/release-playbook.md`
- Templates: `templates/release-checklist.md`, `templates/test-report-template.md`, `templates/changelog-template.md`
